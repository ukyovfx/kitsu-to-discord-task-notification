# v0.2.0 Release Readiness

## 1. Release purpose
v0.2.0 is a Setup Wizard structure and operator clarity release. Its purpose is to make the setup entry path, setup status surface, System Check, Mapping follow-up, and setup/admin page roles easier to understand without changing setup/auth/runtime/API/Docker/DB behavior, changing routes, removing manual setup paths, or adding automation.

## 2. Completed changes since v0.1.3
- Renamed the visible `Quick Setup` UI label to `Setup Status` while keeping the compatible `?mode=quick` route in place
- Aligned current-user docs terminology from `Quick Setup` to `Setup Status`
- Emphasized `Guided Setup` as the recommended first-time path in the setup wizard entry chooser
- Reframed `System Check` as an intentional Guided Setup preflight/status step rather than a broken or hidden Step 0 state
- Reframed Step 4 Mapping as optional post-setup/admin follow-up work instead of part of the core setup finish line
- Clarified role boundaries across `/bot/setup-wizard`, `/bot/admin/setup`, `/bot/setup`, `/bot/admin/bot`, and `/bot/admin/projects`
- Kept existing routes and manual setup/admin surfaces intact

## 3. Validation status
- This release line is limited to Setup Wizard structure, operator wording, and docs clarity work
- No setup/auth/runtime/API/Docker/DB behavior change is intended
- The implementation PRs that changed app/UI copy reported `docker compose config -q` passing where run (#40, #44, #46, #48, #50)
- The implementation PRs that changed app/UI copy reported `docker compose build app` passing where run (#40, #44, #46, #48, #50)
- Docs-only terminology alignment work did not claim build/test execution (#42)
- This readiness note does not claim local `go test` or `go vet` execution
- This readiness note does not claim a runtime or production maturity jump

## 4. Known limitations
- v0.2.0 does not redesign setup architecture beyond wording, guidance, and role clarity
- Manual setup paths remain necessary for some repair, diagnostics, and direct management cases
- Mapping remains optional in presentation, but it can still matter for richer `@mention` routing workflows
- `/bot/setup` remains an active management surface rather than being replaced or removed
- This release improves operator clarity, but does not broaden claims beyond the validated UI/copy/docs scope

## 5. What is intentionally not included
- No setup logic rewrite
- No auth/session change
- No runtime/bootstrap change
- No API/schema change
- No Docker behavior change beyond previously validated app build compatibility
- No DB change
- No route rename or route removal
- No removal of manual setup/admin paths
- No automation or self-healing workflow expansion
- No broader admin dashboard redesign

## 6. Release recommendation
Release now.

Reasoning:
- v0.2.0 is cohesive and single-theme: Setup Wizard structure and operator clarity
- the completed work consistently improves how operators understand first-time setup, status review, preflight blocking, optional follow-up mapping, and setup/admin page boundaries
- the validated scope does not expand into setup/auth/runtime/API/Docker/DB behavior changes

# v0.1.2 Release Readiness

## 1. Release purpose
v0.1.2 is a small setup wizard trust/copy release. Its purpose is to make first-time setup feel safer and easier to understand without changing setup behavior, automation behavior, or runtime behavior.

## 2. Completed changes since v0.1.1
- Added a trust/safety note before Step 3 Discord provisioning confirmation
- Replaced raw Safe to retry: yes/no wording with human-readable retry guidance
- Clarified the purpose of the Kitsu credentials shown in Step 1
- Clarified what the Discord bot needs permission to do in Step 2

## 3. Validation status
- This release line is UI/copy only for the setup wizard
- No setup/auth/runtime/API/Docker/DB behavior change is intended
- Individual v0.1.2 PRs reported docker compose config -q and docker compose build app passing after each copy change
- This readiness note does not claim local go test / go vet execution

## 4. Known limitations
- v0.1.2 improves trust and clarity, but does not redesign the setup wizard
- Rollback remains best-effort; v0.1.2 only clarifies how that should be understood
- Setup credential persistence behavior is unchanged
- Discord permission requirements are unchanged
- The overall setup flow and automation boundaries remain the same as before

## 5. What is intentionally not included
- No setup flow rewrite
- No auth/session change
- No runtime/bootstrap change
- No API/schema change
- No Discord provisioning logic change
- No cleanup automation change
- No new feature scope beyond wizard trust/copy improvements

## 6. Release recommendation
Release now.

Reasoning:
- v0.1.2 is small, consistent, and low-risk
- it improves operator trust and clarity at the most sensitive setup moments
- it does not expand behavior or introduce new setup automation risk

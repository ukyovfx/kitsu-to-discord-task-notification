🎬 DCC Direct Link

Phase 2-2-1: vfx-launcher Integration

✨ What's New:

  • URL Scheme: vfx-launcher://open?software=maya&path=...
  • Supported DCC: Maya 🟢 / Nuke 🔴 / Blender 🟠 / Houdini 🟡 / Katana 🟣
  • Cross-platform path mapping: /mnt/projects → C:\projects (Windows) → /Volumes/projects (macOS)
  • Discord Action Row buttons with automatic file opening
  • Admin UI: /bot/admin/dcc-tools, /bot/admin/path-mapping

📊 Data Tables: ProjectDCCTool, PathMapping, FilePathPreset

⚠️ Error Handling

Phase 2-2-2: Smart Retry Logic

✨ What's New:

  • 429 (Rate Limited) → Respect Retry-After header + exponential backoff
  • 5xx (Server Error) → Exponential backoff: 1s → 2s → 4s (max 3 retries)
  • 4xx (Client Error) → No retry (immediate failure)
  • Auto-detection of transient vs permanent failures
  • Comprehensive error logging for debugging

🎯 Result: Higher success rates, better error transparency

📋 Audit Trail

Phase 2-2-3: Message Audit Log

✨ What's New:

  • AuditLog table: tracks every Discord message sent/failed
  • Recorded data: task_id, status, success/failure, retry_count, error_message, Discord message ID
  • UI: /bot/admin/audit displays last 200 records with timestamps
  • Auto-purge: logs older than 30 days deleted automatically
  • Color-coded display: ✅ Success (green) / ❌ Failure (red)

🔍 Use Case: Troubleshooting, compliance tracking, performance monitoring

🔔 Role Filtering

Phase 2-2-4: Intelligent @Mention Rules

✨ What's New:

  • MentionRule table: TaskType × Status → Discord Role ID mapping
  • Example: "Compositing" + "WFA" (Waiting For Approval) → @Reviewers role
  • Wildcard support: Empty TaskType/Status = "apply to all"
  • Admin UI: /bot/admin/mention-rules for easy rule configuration
  • Precedence: Specific rule > wildcard rule (automatic optimization)

🎯 Result: Targeted notifications, reduced mention spam, team awareness

🗄️ Database Schema — Phase 2-2 New Tables

📋 TABLE: ProjectDCCTool (新規 Phase 2-2-1)
id: INT PRIMARY KEY
kitsu_project_id: STRING (FK → Project)
dcc_type: STRING ('maya', 'nuke', 'blender',...)
file_path_pattern: STRING ('/productions/{project}/{entity}.nk')
entity_type_filter: STRING ('shot', 'asset', 'sequence', or '')
task_type_filter: STRING (optional, e.g. 'Compositing')
enabled: BOOL
📂 TABLE: PathMapping (新規 Phase 2-2-1)
id: INT PRIMARY KEY
kitsu_project_id: STRING (FK → Project)
source_path: STRING ('/mnt/projects/proj_001')
windows_path: STRING ('C:\projects\proj_001')
macos_path: STRING ('/Volumes/projects/proj_001')
enabled: BOOL
📋 TABLE: AuditLog (新規 Phase 2-2-3)
id: INT PRIMARY KEY
kitsu_project_id: STRING
task_id: STRING (Kitsu Task ID)
webhook_url: STRING (destination)
success: BOOL (send成功/失敗)
retry_count: INT (リトライ回数)
error_message: STRING (エラー詳細)
discord_message_id: STRING (送信済みメッセージ ID)
created_at: TIMESTAMP (自動削除: 30日超過)
🔔 TABLE: MentionRule (新規 Phase 2-2-4)
id: INT PRIMARY KEY
kitsu_project_id: STRING (FK → Project)
task_type: STRING ('' = all, 'Compositing', 'Animation',...)
task_status: STRING ('' = all, 'WFA', 'READY', 'DONE',...)
discord_role_id: STRING (@role to mention)
enabled: BOOL
👥 TABLE: UserMap (更新 Phase 2-2-P0)
id: INT PRIMARY KEY
kitsu_project_id: STRING (FK → Project)
kitsu_username: STRING (可変：改名可能)
✨ kitsu_email: STRING (Immutable!)
discord_id: STRING
※ Email がユーザー改名後も有効な ID として機能
🔗 TABLE: ProjectWebhook (更新 Phase 2-2-P0)
id: INT PRIMARY KEY
kitsu_project_id: STRING (FK → Project, UNIQUE)
task_type: STRING ❌ 削除
entity_type: STRING ❌ 削除
webhook_url: STRING (プロジェクト全体で単一 URL)
discord_channel_id: STRING (参照用)
※ 1プロジェクト = 1 Webhook に統一
Message Flow

🔄 Webhook Processing Flow — Phase 2-2 Enhanced

1️⃣ Kitsu Webhook 受信 → /bot/webhook エンドポイント
POST /bot/webhook (Task更新通知)
↓
2️⃣ Task データ抽出 + 認証確認
JWT Token 取得（セッションストア優先）
↓
3️⃣ Status フィルタリング
WFA, READY, DONE 等の重要ステータスのみ処理
↓
4️⃣ Discord メッセージ組立
┌─ Base Embed (既存・必須)
├─ プロジェクト名・Entity・Task型・Status
├─ Status メッセージ + 絵文字
└─ Color: Shot🔵 / Asset🟢 / Sequence🔷
┌─ DCC Action Row ✨ NEW (Phase 2-2-1)
├─ [🔴 Open in Nuke] [🟢 Open in Maya] 等
└─ vfx-launcher:// URL + パスマッピング適用
┌─ @Mention 組立 ✨ NEW (Phase 2-2-4)
├─ 既存: チェッカーメンション
├─ 既存: アーティストメンション
└─ ✨ 新規: MentionRule ロール自動メンション
└─ Meta 情報 (GoogleDrive URL, DCC情報等)
↓
5️⃣ Webhook 送信（リトライロジック ✨ NEW Phase 2-2-2）
初回送信試行
├─ ✅ 2xx → 成功・ログ記録
├─ 429 → Retry-After 待機 + 指数バックオフリトライ
├─ 5xx → 指数バックオフ (1s→2s→4s) 最大3回
└─ 4xx → リトライなし・即失敗
↓
6️⃣ 監査ログ記録 ✨ NEW (Phase 2-2-3)
AuditLog テーブルに記録
├─ task_id, success/failure, retry_count
├─ error_message (失敗時の詳細)
└─ discord_message_id (送信済みメッセージ参照)
↓
✅ 完了 → アーティストが Discord で通知受け取り
DCC ボタンクリック → ローカルランチャー起動 → DCC+ファイルオープン

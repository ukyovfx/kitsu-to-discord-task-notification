# KitsuSync Release Notes Sync

This directory keeps repo-local copies of release-critical notes that were originally tracked in Obsidian.

For v0.1.0, the canonical repo-facing set is:

- `KitsuSync-CURRENT-STATE.md`
- `KitsuSync-Technical-Spec.md`
- `KitsuSync-v0.1.0-Release-Readiness.md`
- `KitsuSync-v0.1.0-Release-Gate.md`

All four documents use the same status vocabulary:

- `done`: in v0.1.0 scope and implemented
- `partial`: implemented but still needs verification, docs, or smoke coverage
- `v0.2.0`: intentionally out of v0.1.0 scope and moved to roadmap
- `human-check`: requires manual verification with real infrastructure

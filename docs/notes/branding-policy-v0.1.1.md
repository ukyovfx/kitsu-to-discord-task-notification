# Branding Policy for v0.1.1

## Context
KitsuSync is now the public product and project name. The repository still uses the historical slug kitsu-to-discord-task-notification, and the history also reflects upstream or fork lineage.

## Current naming state
- Public product name: KitsuSync
- Current repository slug: ukyovfx/kitsu-to-discord-task-notification
- Upstream attribution remains relevant for history, license, and provenance

## Decision for v0.1.1
- Keep the repository name unchanged during v0.1.1.
- Treat KitsuSync as the public-facing product and project name.
- Treat kitsu-to-discord-task-notification as the historical repository slug.
- Use KitsuSync consistently in public docs and release messaging, while preserving the correct clone URL and repo references.
- Keep upstream/fork attribution visible where license or provenance matters.

## Rationale
- Renaming the repository during v0.1.1 adds avoidable risk to clone URLs, docs, links, releases, and user expectations.
- The public naming problem can be handled in docs and release messaging without changing repository infrastructure.
- Attribution and license clarity are more important than hiding the repository's inherited history.

## Future rename conditions
Consider a repository rename only after a separate issue or decision that evaluates:
- clone URL impact
- existing docs and release link impact
- GitHub redirect behavior and user expectations
- attribution and license visibility after rename

## What not to do now
- Do not rename the repository in v0.1.1.
- Do not change GitHub repo settings as part of this policy note.
- Do not present the historical slug as the public product name.
- Do not hide upstream or fork history where attribution is required.

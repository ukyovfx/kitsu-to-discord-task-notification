# v0.2.1 Release Readiness

## 1. Release purpose
v0.2.1 is a repository rename and public URL alignment release. Its purpose is to align the repository slug with the public product name KitsuSync, update current public clone URLs and operator-facing repository references, and document the rename decision without changing application behavior or setup/auth/runtime/API/Docker/DB behavior.

## 2. Completed changes since v0.2.0
- Renamed the GitHub repository from `ukyovfx/kitsu-to-discord-task-notification` to `ukyovfx/kitsusync`
- Updated the local `origin` remote used in the real working repo to `git@github.com:ukyovfx/kitsusync.git`
- Updated current public clone URLs and `cd` targets in `README.md`, `docs/QUICK_START.md`, and `docs/SETUP_FOR_STUDIOS.md`
- Added `docs/notes/repo-rename-v0.2.1.md` to document the rename decision, canonical repository URLs, redirect guidance, and local clone follow-up
- Kept historical notes and upstream provenance references intact rather than rewriting history

## 3. Validation status
- This release line is limited to repository rename follow-up, canonical URL alignment, and docs updates
- No application behavior change is intended in v0.2.1
- No setup/auth/runtime/API/Docker/DB behavior change is intended
- #56 validated the docs and remote update scope by checking `git remote -v`, `git status --short --untracked-files=all`, `git diff --stat`, `git diff`, and old-slug grep results after the rename follow-up
- #56 confirmed remaining old-slug references were historical only after the current public clone URLs were updated
- This readiness note does not claim local `go test` or `go vet` execution
- This readiness note does not claim Docker build/test execution
- This readiness note does not claim a runtime or production maturity jump

## 4. Known limitations
- v0.2.1 does not change setup, login, admin, mapping, or notification behavior
- Older GitHub URLs may still redirect, but redirects should not be treated as the long-term canonical path in docs or operator instructions
- Historical release notes and policy notes still mention the previous repository slug where they accurately describe earlier project state
- This release improves repository naming consistency, but does not broaden product claims beyond rename and docs alignment work

## 5. What is intentionally not included
- No setup flow rewrite
- No auth/session change
- No runtime/bootstrap change
- No API/schema change
- No Docker behavior change
- No DB change
- No UI change beyond previously merged docs/repository reference updates
